# Software Design & Engineering Enhancement

## Original Artifact

The artifact selected for this enhancement is the **CS-465 Travlr Getaways Full-Stack Web Application**. This project was originally developed as a MEAN stack application consisting of a client-facing travel booking interface and an administrative backend API. The system included route handling, API endpoints, MongoDB integration, and server-side logic.

The original version demonstrated full-stack functionality but required improvements in security implementation, configuration management, and architectural organization.

---

## Enhancements Implemented

The enhanced version strengthens the application through several software engineering improvements:

- Implemented **JSON Web Token (JWT) authentication**
- Added **bcrypt password hashing**
- Secured protected API routes using middleware verification
- Removed hardcoded credentials
- Implemented **environment variable configuration**
- Improved project structure and separation of concerns
- Enhanced route protection and admin authorization
- Improved documentation and code clarity

These improvements demonstrate growth in secure software development practices and architectural refinement.

---

## Technical Design Improvements

The updated architecture now demonstrates:

- Secure authentication workflow using JWT
- Middleware-based authorization checks
- Encrypted password storage
- Environment-based configuration using `.env`
- Clean separation between API logic and client logic
- Improved modular route handling

Security improvements were validated using Thunder Client to test authentication, token generation, and protected route access.

These changes reflect industry-standard practices for full-stack web application development.

---

## Reflection

Through this enhancement, I strengthened my understanding of secure software design principles, authentication workflows, and backend architecture. Refactoring the application to remove hardcoded credentials and implement environment variables improved maintainability and scalability.

The implementation of JWT authentication and encrypted password storage aligns with professional development standards and demonstrates competency in software security and system design.

This artifact directly supports Computer Science program outcomes related to secure coding practices, software lifecycle management, and professional software engineering standards.
