# Databases Enhancement

## Original Artifact

The original artifact selected for this enhancement is the CS-340 Grazioso Salvare AnimalShelter CRUD module. The project involved building a Python class using PyMongo to connect to a MongoDB database containing animal shelter data. The system supported basic Create and Read functionality and was integrated with a Jupyter Notebook dashboard to display filtered results.

The initial submission demonstrated functional database connectivity but required improvements in security, modular design, and expanded CRUD capabilities.

---

## Enhancements Made

The enhanced version improves security, structure, and database functionality through:

- Implementation of full CRUD operations (Create, Read, Update, Delete)
- Removal of hardcoded database credentials
- Integration of environment variable configuration for secure authentication
- Improved query validation and filtering logic
- Refactored class structure for better modularity
- Enhanced exception handling for database operations
- Improved documentation and code readability

These improvements increase maintainability, scalability, and security of the application.

---

## Technical Details

The enhanced system uses:

- MongoDB for document-based data storage
- PyMongo for database connectivity
- Object-Oriented Python design
- Structured CRUD methods with parameterized queries
- Environment variable configuration to protect sensitive credentials
- Filter-based queries using JSON-style criteria

Update and delete operations were implemented to complete full CRUD functionality. Query refinement improved performance when retrieving filtered animal records. The removal of hardcoded credentials aligns the project with secure development best practices.
